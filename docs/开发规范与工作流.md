## 一、项目结构

```
Neko_Status/
├── src/
│   ├── main/
│   │   └── main.js            # Electron 主进程（窗口管理、IPC、系统API）
│   └── renderer/
│       ├── index.html         # 页面骨架，只写 DOM 结构，不写内联 style/script
│       ├── css/
│       │   └── main.css       # 全局样式、设计 Token、主题变量
│       └── js/
│           └── app.js         # 渲染进程逻辑（路由、事件、主题、布局）
├── assets/                    # 图标、图片等静态资源
├── docs/                      # 项目文档（本文件所在目录）
├── package.json               # 项目元信息，name=neko-status-desktop
└── README.md
```

### 三层严格职责分工

| 文件         | 只允许写什么                         | 绝对禁止                           |
| ------------ | ------------------------------------ | ---------------------------------- |
| `index.html` | DOM 节点、`id`/`class`/`data-*` 属性 | 内联 `style="..."`、`<script>` 块  |
| `main.css`   | CSS 变量、规则、动画                 | JavaScript、内联逻辑               |
| `app.js`     | DOM 操作、事件绑定、状态管理         | 硬编码颜色/尺寸（应改写 CSS 变量） |

## 二、日常开发命令

```powershell
# 进入项目根目录
cd "d:\VScode project\Neko_Status"

# 启动开发模式（含 DevTools）
npm start
```

### 热重载（推荐开发阶段常用）

**渲染进程（HTML / CSS / JS）修改后 — 无需重启进程，直接刷新页面：**

在 Electron 窗口内按 `Ctrl + R` 即可热刷新渲染进程，CSS 和 JS 改动立即生效。

**主进程（`main.js`）修改后 — 需重启 Electron 进程：**

```powershell
# 一键热重载：杀掉旧进程后立即重启（在项目根目录执行）
Get-Process -Name "electron" -ErrorAction SilentlyContinue | Stop-Process -Force; Start-Sleep -Milliseconds 500; npm start
```

> ✅ 开发原则：**优先使用 `Ctrl + R` 热刷新**，只有修改了 `main.js` 时才执行完整重启。避免频繁冷启动拖慢迭代节奏。

### DevTools 调试面板

Electron 开发模式下默认打开 DevTools。可在 DevTools Console 中临时执行 CSS 变量修改或 JS 调试，但**确认方案后必须同步写入 `main.css` / `app.js`**，不能只留在 DevTools 里。

```javascript
// DevTools Console 快速预览主题色（仅用于调试，不提交）
document.documentElement.style.setProperty("--theme-color", "#f59e0b");
```

## 三、新增页面/模块的标准步骤

### 步骤 1：在 `index.html` 中添加 DOM 骨架

```html
<!-- 新页面容器。默认隐藏，由 JS 路由控制显示 -->
<div class="page" id="page-settings" style="display:none;">
  <div class="content-safe-area">
    <!-- 页面内容 -->
  </div>
</div>
```

- 每个页面容器添加 class `page` 和唯一 `id="page-xxx"`
- 默认 `display: none`，由 `app.js` 的路由函数统一切换

### 步骤 2：在 `main.css` 中添加页面专属样式

```css
/* ========== 设置页 ========== */
.settings-section {
  /* ... */
}
```

- 样式从不内联写在 HTML 中
- 颜色一律使用 CSS 变量（`var(--theme-color)`、`var(--text-main)` 等）
- 新增语义变量在 `:root {}` 块中统一声明

### 步骤 3：在 `app.js` 中注册路由

```javascript
// 在路由表 navItems 中添加对应条目
const navItems = [
  { btnId: "navSettings", pageId: "page-settings" },
  // ...
];
```

## 四、主题与设计系统规范

### CSS 变量（Design Tokens）

所有颜色、间距、圆角均使用 CSS 变量，**不允许硬编码**：

```css
/* 正确 ✅ */
background: var(--glass-bg);
color: var(--theme-color);
border-radius: var(--radius-card);

/* 错误 ❌ */
background: rgba(255, 255, 255, 0.05);
color: #06b6d4;
border-radius: 16px;
```

### 主题色切换

主题色通过 JS 动态修改 CSS 根变量来实现，支持暗色/亮色双模式：

```javascript
// 切换主题色
document.documentElement.style.setProperty("--theme-color", "#f59e0b");
localStorage.setItem("neko-theme-color", "#f59e0b");

// 切换深浅模式
document.documentElement.setAttribute("data-theme", "light"); // 或 'dark'
localStorage.setItem("neko-theme-mode", "light");
```

## 五、网格布局规范（Dashboard Grid）

当前仪表盘使用 **12 列 × N 行**的高精度网格系统：

```css
.dashboard-section {
  display: grid;
  grid-template-columns: repeat(12, minmax(65px, 1fr));
  grid-auto-rows: 40px;
  gap: 16px;
}
```

### 卡片宽高标准单位

| data-w / data-h | 实际含义          | 建议用途   |
| --------------- | ----------------- | ---------- |
| `data-w="3"`    | 占 3 列           | 小型指标卡 |
| `data-w="6"`    | 占 6 列（半宽）   | 图表、列表 |
| `data-w="9"`    | 占 9 列（3/4 宽） | 大型图表   |
| `data-w="12"`   | 占全宽            | 横向信息流 |
| `data-h="3"`    | 3 × 40 = 120px 高 | 指标       |
| `data-h="6"`    | 6 × 40 = 240px 高 | 图表       |

### 布局持久化

布局变更通过 `localStorage.setItem('neko_layout_config', JSON.stringify(layout))` 持久化，下次启动自动还原。

## 八、禁止行为清单

- ❌ 将新功能直接写进 `index.html` 的 `<script>` 标签
- ❌ 使用 `document.write()` 或 `.innerHTML` 直接拼接 HTML 字符串
- ❌ 在 JS 中硬编码颜色值（应修改 CSS 变量）
- ❌ `overflow: hidden` 用在网格父容器上（会截断卡片阴影）
- ❌ FLIP 动效完成后不清除内联 `transition` 属性（会导致布局入侵）
- ❌ 多次重复声明同一个 `const` 变量（会导致 JS 引擎抛出引用错误冻结整个渲染进程）
- ❌ 给元素写内联 `style="font-size/color/..."` 硬编码尺寸或颜色（应全部提取到 `main.css` 对应类名中）
- ❌ 新增内联 `style` 来调整字体大小、位置或颜色（差异状态用专用修饰类，如 `.kpi-value-sm`）

## 九、页面路由切换规范

> ⚠️ 已踩坑：用 JS `element.style.display = 'block'` 显示 `.content-safe-area` 节点，会覆盖 CSS 类的 `display: flex`，导致内部网格卡片相互挤压。

### 正确做法

```javascript
// ✅ 正确：根据元素的 CSS display 类型恢复对应值
const FLEX_AREAS = ["mainDashboardArea", "consoleArea"];

function showArea(id) {
  const el = document.getElementById(id);
  if (!el) return;
  el.style.display = FLEX_AREAS.includes(id) ? "flex" : "block";
}
```

| 区域 ID                      | 应该使用的 display                             |
| ---------------------------- | ---------------------------------------------- |
| `mainDashboardArea`          | `flex`（`.content-safe-area` CSS 类要求）      |
| `consoleArea`                | `flex`（`.content-safe-area` CSS 类要求）      |
| `page-xxx`（外层 page 容器） | `block`（内层 `.content-safe-area` 自带 flex） |

## 十、卡片内容视觉层级规范

卡片内部必须严格遵循三层视觉层级，从大到小依次排列：

```
【第一层 最大】  标题文字 + 附属图标    →  18px  font-weight: 700  color: text-main
【第二层 次之】  主要内容（数值/状态）  →  15px  font-weight: 600  color: text-secondary
【第三层 最小】  次要说明（备注/单位）  →  12px  font-weight: 400  color: text-secondary
```

> ❌ **典型错误**：让主值 `30px` 远大于标题 `14px`——用户第一眼看到的是"某个数字"而不是"这是哪个指标"，破坏信息可读性。

### 对应 CSS 类

| 类名                             | 层级   | 规格                                 |
| -------------------------------- | ------ | ------------------------------------ |
| `.metric-header` / `.kpi-header` | 第一层 | 18px 700 text-main                   |
| `.metric-icon` / `.kpi-header i` | 第一层 | 20-22px，主题色                      |
| `.metric-value` / `.kpi-value`   | 第二层 | 15px 600 text-secondary              |
| `.kpi-value-sm`                  | 第二层 | 与 `.kpi-value` 相同，用于带单位变体 |
| `.kpi-value-unit`                | 第三层 | 13px 400，数值后单位文字             |
| `.metric-trend` / `.kpi-footer`  | 第三层 | 12px text-secondary                  |

### 示例对比

```html
<!-- ✅ 正确：标题最大 → 值次之 → 备注最小 -->
<div class="kpi-header">
  <span><i class="ph ph-cpu"></i> CPU 负载</span>
  <span class="kpi-badge info">正常</span>
</div>
<div class="kpi-value-row">
  <div class="kpi-value">12.4<small>%</small></div>
</div>
<div class="kpi-footer">温度: 45°C / 频率: 3.2GHz</div>

<!-- ❌ 错误：数值比标题大，层级倒置 -->
<div style="font-size:14px">CPU 负载</div>
<div style="font-size:30px; font-weight:bold">12.4%</div>
<div style="font-size:12px">温度: 45°C</div>
```

## 十一、开关（Toggle Switch）规范

> ⚠️ 已踩坑：曾使用 `<input type="checkbox"> + <label class="toggle-switch-track">` 方案配合 CSS `:checked` 伪类控制开关动画，导致第二个 `.toggle-switch { display: flex }` CSS 规则覆盖了正确的绝对定位实现，白点始终不动。

### 唯一允许的开关方案（与仪表盘完全一致）

**HTML：**

```html
<!-- 初始关闭 -->
<div class="toggle-switch" id="mySwitch"></div>
<!-- 初始开启 -->
<div class="toggle-switch on" id="mySwitch"></div>
```

**JS（点击切换 + 状态读取）：**

```javascript
const sw = document.getElementById("mySwitch");
sw.addEventListener("click", () => sw.classList.toggle("on"));

// 读取状态
const isOn = sw.classList.contains("on");
```

**CSS（已在 :root 附近定义，禁止再次声明 .toggle-switch 基础规则）：**

```css
.toggle-switch { position: relative; width: 44px; height: 24px; ... }
.toggle-switch::after { position: absolute; top: 1px; left: 2px; width: 20px; height: 20px; ... }
.toggle-switch.on { background: var(--theme-color); }
.toggle-switch.on::after { transform: translateX(20px); }
```

### 禁止行为

- ❌ **绝对禁止** 使用 `<input type="checkbox"> + <label>` 配合 `:checked` 实现开关
- ❌ 在任何位置重复声明 `.toggle-switch { display: flex }` 或覆盖 `position: relative`
- ❌ 用 JS `.checked` 属性读取 div-based 开关状态（应用 `.classList.contains('on')`）
- ❌ 监听 `'change'` 事件（div 无此事件，应监听 `'click'`）

## 十二、筛选器/分段控件规范

对于"全部/正常/警告/错误"等类别筛选，使用 **分段滑块（Segmented Control）** 组件，而非普通 `toggle-btn` 组：

```html
<!-- ✅ 正确：分段滑块筛选器 -->
<div class="filter-segmented" id="myFilterGroup">
  <div class="filter-segmented-pill" id="myFilterPill"></div>
  <button class="filter-segmented-btn active" data-filter="all">全部</button>
  <button class="filter-segmented-btn" data-filter="success">正常</button>
  <button class="filter-segmented-btn" data-filter="warn">警告</button>
  <button class="filter-segmented-btn" data-filter="error">错误</button>
</div>
```

- 被筛选的数据行必须标记 `data-status="success|warn|error"` 属性
- JS 负责滑块位置同步 (`syncFilterPill`) 与行显示/隐藏（配合 `tableRowFadeIn` 动画）
- 每种状态下激活按钮颜色通过 `.filter-segmented-btn[data-filter="warn"].active { color: var(--warning-amber); }` 语义化
