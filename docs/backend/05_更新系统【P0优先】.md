# 05 — 更新系统设计文档【P0 优先级】

> **说明**：软件处于 Beta 阶段。确保用户能正常接收更新、安装更新是当前最高优先级。  
> 本文档覆盖：GitHub 自动发版 CI/CD → 后端版本服务 → 客户端检测/下载/安装 的完整链路。

## 整体流程图

```
开发者推送 Tag
     │
     ▼
GitHub Actions
  ├── 构建 Electron 应用（Windows）
  ├── 生成 .exe 安装包 + .zip 压缩包
  ├── 计算 SHA256 校验值
  └── 创建 GitHub Release（自动发布）
                │
                ▼ （定时同步 / Webhook）
          Neko 后端服务
          update_releases 表
                │
          ┌─────┴─────┐
          │           │
     客户端轮询    WebSocket 推送
  GET /update/check  update_available
          │
          ▼
      用户确认更新
          │
    ┌─────┴──────┐
    │            │
  自动下载      手动安装
  .exe 安装包   .zip / .7z 文件
    │            │
    └─────┬──────┘
          │
     SHA256 验证
          │
     执行安装程序
```

## 一、GitHub Actions — 自动构建并发布

### 文件路径

`.github/workflows/release.yml`（放在 Electron 客户端仓库根目录）

### 工作流配置

```yaml
name: Release

on:
  push:
    tags:
      - "v*.*.*" # 触发：推送类似 v1.1.0 的 tag
      - "v*.*.*-beta.*" # 触发：推送类似 v1.1.0-beta.1 的 tag

permissions:
  contents: write # 需要写权限才能创建 Release

jobs:
  build-windows:
    runs-on: windows-latest

    steps:
      - name: Checkout 代码
        uses: actions/checkout@v4

      - name: 安装 Node.js
        uses: actions/setup-node@v4
        with:
          node-version: "20"
          cache: "npm"

      - name: 安装依赖
        run: npm ci

      - name: 构建 Electron 应用
        run: npm run build
        # 实际上会调用 electron-builder，产出 dist/ 目录

      - name: 计算校验值
        run: |
          $version = "${{ github.ref_name }}" -replace '^v',''
          $exeFile = "dist/NekoStatus-Setup-${version}.exe"
          $zipFile = "dist/NekoStatus-${version}-win.zip"

          $exeHash = (Get-FileHash $exeFile -Algorithm SHA256).Hash.ToLower()
          $zipHash = (Get-FileHash $zipFile -Algorithm SHA256).Hash.ToLower()

          # 写入校验文件
          "${exeHash}  NekoStatus-Setup-${version}.exe" | Out-File "dist/SHA256SUMS.txt" -Encoding utf8
          "${zipHash}  NekoStatus-${version}-win.zip" | Out-File "dist/SHA256SUMS.txt" -Append -Encoding utf8

          echo "EXE_SHA256=$exeHash" >> $env:GITHUB_ENV
          echo "ZIP_SHA256=$zipHash" >> $env:GITHUB_ENV
          echo "VERSION=$version" >> $env:GITHUB_ENV
        shell: pwsh

      - name: 创建 GitHub Release 并上传产物
        uses: softprops/action-gh-release@v2
        with:
          name: "Neko Status v${{ env.VERSION }}"
          prerelease: ${{ contains(github.ref_name, '-beta') || contains(github.ref_name, '-nightly') }}
          generate_release_notes: true # 自动从 commit 生成 changelog
          files: |
            dist/NekoStatus-Setup-${{ env.VERSION }}.exe
            dist/NekoStatus-${{ env.VERSION }}-win.zip
            dist/SHA256SUMS.txt
          body: |
            ## 安装说明

            - **推荐**：下载 `NekoStatus-Setup-${{ env.VERSION }}.exe` 双击安装
            - **手动**：下载 `.zip` 解压后覆盖安装

            ### 文件完整性校验

            | 文件 | SHA256 |
            |------|--------|
            | NekoStatus-Setup-${{ env.VERSION }}.exe | `${{ env.EXE_SHA256 }}` |
            | NekoStatus-${{ env.VERSION }}-win.zip | `${{ env.ZIP_SHA256 }}` |
```

## 三、后端版本服务

### 3.1 同步 GitHub Releases

后端需要定时将 GitHub Releases 数据同步到本地 `update_releases` 表（避免每次检查都打 GitHub API，同时防止因 GitHub 故障影响更新体验）。

**同步逻辑（伪代码）**

```javascript
// jobs/syncReleases.js
const GITHUB_API = "https://api.github.com/repos/{owner}/{repo}/releases";

async function syncReleases(githubRepo, channelId) {
  const response = await fetch(
    `https://api.github.com/repos/${githubRepo}/releases`,
    {
      headers: {
        Authorization: `Bearer ${process.env.GITHUB_TOKEN}`,
        Accept: "application/vnd.github+json",
      },
    },
  );

  const releases = await response.json();

  for (const release of releases) {
    // 找到 .exe 和 .zip 产物
    const exeAsset = release.assets.find((a) => a.name.endsWith(".exe"));
    const zipAsset = release.assets.find((a) => a.name.endsWith("-win.zip"));
    const sumsAsset = release.assets.find((a) => a.name === "SHA256SUMS.txt");

    let exeSha256 = null,
      zipSha256 = null;
    if (sumsAsset) {
      const sumsText = await fetchText(sumsAsset.browser_download_url);
      exeSha256 = parseSHA256(sumsText, ".exe");
      zipSha256 = parseSHA256(sumsText, ".zip");
    }

    // 确定通道
    const channel = release.prerelease
      ? release.tag_name.includes("beta")
        ? "beta"
        : "nightly"
      : "stable";

    // 同步到数据库
    await db.upsert("update_releases", {
      channel_id: channel,
      version: release.tag_name.replace(/^v/, ""),
      tag_name: release.tag_name,
      release_date: new Date(release.published_at),
      changelog: release.body,
      win_exe_url: exeAsset?.browser_download_url,
      win_exe_sha256: exeSha256,
      win_exe_size: exeAsset?.size,
      win_zip_url: zipAsset?.browser_download_url,
      win_zip_sha256: zipSha256,
      win_zip_size: zipAsset?.size,
      github_release_id: release.id,
      synced_at: new Date(),
    });
  }

  // 更新 is_latest 标记
  await markLatestRelease(channelId);
}

// 定时任务：每小时同步一次
setInterval(() => syncReleases(), 60 * 60 * 1000);
```

### 3.2 `GET /api/v1/update/check` 实现逻辑

```javascript
// routes/update.js
router.get("/check", authMiddleware, async (req, res) => {
  const { version, channel = "stable" } = req.query;

  if (!version) {
    return res
      .status(400)
      .json({
        ok: false,
        error: { code: "BAD_REQUEST", message: "缺少 version 参数" },
      });
  }

  // 查询最新版本
  const latest = await db.query(
    `SELECT * FROM update_releases 
     WHERE channel_id = $1 AND is_latest = TRUE AND win_exe_url IS NOT NULL
     LIMIT 1`,
    [channel],
  );

  if (!latest) {
    return res.json({ ok: true, data: { has_update: false } });
  }

  const hasUpdate = semver.gt(latest.version, version);

  // 记录检查日志
  await logUpdateCheck(req.device_id, version, channel, hasUpdate);

  return res.json({
    ok: true,
    data: {
      has_update: hasUpdate,
      current_version: version,
      latest_version: latest.version,
      channel,
      release_date: latest.release_date,
      download_url: latest.win_exe_url,
      checksum_sha256: latest.win_exe_sha256,
      file_size_bytes: latest.win_exe_size,
      zip_url: latest.win_zip_url,
      zip_sha256: latest.win_zip_sha256,
      changelog: latest.changelog,
      is_forced: latest.is_forced,
    },
  });
});
```

## 四、客户端更新流程（前端实现参考）

> 仅作为实现指引，Electron 客户端开发者参考此流程对接后端 API。

### 4.1 自动检查更新

```javascript
// 客户端 update-manager.js（参考实现）
const { app, shell } = require("electron");
const { download } = require("electron-dl"); // 或使用 Node.js 原生 https

async function checkForUpdates() {
  const currentVersion = app.getVersion();
  const channel = settings.get("updateChannel") || "stable";
  const serverUrl = settings.get("serverUrl");

  const res = await fetch(
    `${serverUrl}/api/v1/update/check?version=${currentVersion}&channel=${channel}`,
    {
      headers: { "X-API-Key": settings.get("apiKey") },
    },
  );

  const { data } = await res.json();

  if (data.has_update) {
    if (data.is_forced) {
      // 强制更新：弹出不可关闭的更新弹窗
      showForceUpdateDialog(data);
    } else {
      // 普通更新：右上角徽标提示
      updateBadge.show(data.latest_version);
    }
  }
}

// 每 6 小时检查一次
setInterval(checkForUpdates, 6 * 60 * 60 * 1000);
app.whenReady().then(() => setTimeout(checkForUpdates, 5000)); // 启动 5s 后检查
```

### 4.2 下载并安装更新

```javascript
async function downloadAndInstall(updateInfo) {
  const { download_url, checksum_sha256, file_size_bytes } = updateInfo;

  // 1. 显示下载进度
  setDownloadProgress(0);

  // 2. 下载到临时目录
  const tempPath = path.join(os.tmpdir(), "NekoStatus-update.exe");
  await downloadFile(download_url, tempPath, (progress) => {
    setDownloadProgress(progress); // 0~100
  });

  // 3. 验证 SHA256
  const actualHash = await computeSHA256(tempPath);
  if (actualHash !== checksum_sha256) {
    throw new Error("文件完整性校验失败，下载可能已损坏，请重试");
  }

  // 4. 执行安装（此操作会关闭当前应用）
  const { spawn } = require("child_process");
  spawn(tempPath, ["/S"], {
    // /S = 静默安装（NSIS 参数）
    detached: true,
    stdio: "ignore",
  }).unref();

  app.quit();
}
```

### 4.3 手动安装（本地 .zip / .7z 文件）

> 用户可在"更新中心"手动选择本地安装包文件（适用于离线环境或增量更新）。

```javascript
// 客户端处理本地安装包
async function installFromLocalFile(filePath) {
  const ext = path.extname(filePath).toLowerCase();

  if (ext === ".exe") {
    // 直接执行安装程序
    shell.openPath(filePath);
    return;
  }

  if (ext === ".zip") {
    // 解压到应用目录（覆盖安装）
    const installDir = path.dirname(app.getPath("exe"));

    // 使用 node-stream-zip 或 AdmZip 解压
    const zip = new AdmZip(filePath);

    // 关闭前备份当前版本（用于回滚）
    await backupCurrentVersion(installDir);

    // 解压覆盖
    zip.extractAllTo(installDir, /*overwrite*/ true);

    // 重启应用
    app.relaunch();
    app.quit();
    return;
  }

  throw new Error(`不支持的文件格式：${ext}，请使用 .exe 或 .zip 文件`);
}
```

### 5.1 回滚逻辑

后端 `update_releases` 表中保留所有历史版本记录，客户端可请求获取"上一个版本"的下载链接。

## 六、通道管理

| 通道      | Tag 格式                  | 说明                       | 推荐用户       |
| --------- | ------------------------- | -------------------------- | -------------- |
| `stable`  | `v1.0.0`                  | 正式发布，全量测试通过     | 所有用户       |
| `beta`    | `v1.1.0-beta.1`           | 新功能预览，可能有已知问题 | 愿意测试的用户 |
| `nightly` | 自动构建（每日/每次提交） | 最新代码，不保证稳定       | 开发者         |

**GitHub Tag 命名规范：**

```bash
# 正式版
git tag v1.0.0
git push origin v1.0.0

# Beta 版
git tag v1.1.0-beta.1
git push origin v1.1.0-beta.1

# 无需手动打 tag，Actions 定时触发构建 Nightly
```

## 七、用户自定义更新源

> 客户端支持用户配置自定义 GitHub 仓库 URL 作为更新源（在"更新设置"中配置）。

**后端处理逻辑：**

1. 客户端在检查更新时，可传入 `source_repo` 参数
2. 后端验证 GitHub 仓库 URL 合法性（防止 SSRF）
3. 代理向该仓库查询最新 Release 信息并返回
4. 白名单策略：只允许 `github.com/{owner}/{repo}` 格式

**SSRF 防护（重要）：**

```javascript
// 验证用户提供的 GitHub 仓库 URL
function validateGithubRepo(repo) {
  // 只允许 格式：owner/repo，不允许完整 URL
  const pattern = /^[a-zA-Z0-9_.-]+\/[a-zA-Z0-9_.-]+$/;
  if (!pattern.test(repo)) {
    throw new Error("仓库格式无效，请使用 owner/repo 格式");
  }
  // 最终拼接为固定的 API URL，不允许用户控制域名
  return `https://api.github.com/repos/${repo}/releases/latest`;
}
```

## 八、强制更新机制

使用场景：发现严重安全漏洞、协议不兼容时，后端标记某版本范围为"强制更新"。

**后端操作：**

```sql
-- 将 1.0.0 以下版本标记为强制更新目标（需要升级到最新）
UPDATE update_releases SET is_forced = TRUE
WHERE channel_id = 'stable' AND version = (SELECT MAX(version) FROM update_releases WHERE channel_id = 'stable');
```

**客户端行为：**

- 收到 `is_forced: true` 时，弹出不可关闭的强制更新弹窗
- 或通过 WebSocket `force_update` 消息实时下发（见 03 文档）

## 九、快速实施路线图

### Beta 阶段最小可行版本（MVP）

```
阶段一（必须完成）：
  ✅ GitHub Actions 自动构建 + 创建 Release
  ✅ GET /api/v1/update/check — 返回最新版本信息
  ✅ 客户端检查更新 + 跳转 GitHub 下载页

阶段二（尽快完成）：
  ⬜ auto-download：客户端内直接下载 .exe
  ⬜ SHA256 验证
  ⬜ 自动执行安装程序

阶段三（完善）：
  ⬜ 手动 .zip 本地安装
  ⬜ 版本回滚
  ⬜ 通道切换（stable/beta）
  ⬜ WebSocket force_update 推送
```

> **Beta 阶段最低要求**：当用户打开"更新中心"，能看到当前版本与最新版本，并能点击按钮跳转到 GitHub Releases 页面下载安装。
