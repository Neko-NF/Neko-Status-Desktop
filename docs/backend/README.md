## 当前阶段说明（Beta）

**当前处于 Beta 迭代阶段，后端实现优先级如下：**

```
P0（阻塞性功能，必须先做）
├── 更新系统（见文档 05）—— 客户端必须能收到更新并完成安装
├── 设备认证 + API Key 鉴权（见文档 06）
└── 核心上报接口（见文档 02：设备状态、截图上传）

P0.5（v1.1.0 新增功能，当前迭代重点）
├── SRS 直播推流接口（见文档 02 第六节）
│   ├── Stream Key 生成/重置（GET/POST /api/v1/stream/key）
│   ├── SRS 连通性代理测试（POST /api/v1/stream/test-srs）
│   └── 推流状态代理查询（GET /api/v1/stream/status）
├── 推流数据库表（见文档 04：device_stream_keys）
└── Stream Key 安全管理（见文档 06 第九节）

P1（Beta 阶段内完成）
├── WebSocket 实时推送
├── 历史记录查询
└── 健康度统计

P2（后续版本）
├── SRS on_publish 回调记录直播会话（v1.2.0）
├── 多设备管理
├── 告警通知
└── 配置云同步
```

> 📌 **v1.1.0 关联文档**：《SRS直播推流-产品需求与功能规划.md》、《SRS直播推流-后端实现方案.md》、《SRS直播推流-开发任务分配与交付标准.md》

## 快速链接

- 客户端服务器地址配置：`设置 → 数据与上传 → 服务器地址`，默认 `https://api.koirin.com/neko`
- 本地测试模式：客户端支持切换到 `http://localhost:8080`，无需 API Key
- WebSocket 端点：`wss://{host}/neko/ws`
