## 二、技术选型建议

> 以下为推荐方案，后端团队可根据实际情况调整。

### 推荐：Node.js + Express / Fastify

```
优点：
- 与 Electron 主进程语言一致，团队上手快
- 生态丰富，multipart 上传、WebSocket 库成熟
- 与 GitHub Actions 发布自动化配合方便

依赖清单（参考）：
express / fastify       — HTTP 框架
ws / socket.io          — WebSocket
multer                  — 文件上传
better-sqlite3 / pg     — 数据库驱动
jsonwebtoken            — 可选：JWT 鉴权
node-cron               — 定时任务（轮询 GitHub Releases）
```

### 备选：Python + FastAPI

```
uvicorn          — ASGI 服务器
fastapi          — 路由框架
sqlalchemy       — ORM
aiofiles         — 异步文件操作
websockets       — WS 支持
```

## 四、环境变量（`.env.example`）

```dotenv
# 服务端口
PORT=8080

# 数据库
DATABASE_URL=sqlite:./neko.db
# 或 PostgreSQL:
# DATABASE_URL=postgresql://user:pass@localhost:5432/neko_status

# API Key 加盐（随机32位十六进制字符串）
API_KEY_SECRET=your_random_secret_here

# GitHub 更新源（默认值，客户端可覆盖）
GITHUB_DEFAULT_REPO=your-org/neko-status
GITHUB_TOKEN=ghp_xxxxxxxxxxxx   # 可选，提升 GitHub API 速率限制

# 文件存储
UPLOAD_DIR=./uploads
MAX_UPLOAD_SIZE_MB=10

# WebSocket 心跳间隔（秒）
WS_HEARTBEAT_INTERVAL=30
```

## 五、本地开发快速启动

```bash
# 1. 克隆仓库
git clone https://github.com/your-org/neko-backend.git
cd neko-backend

# 2. 安装依赖
npm install

# 3. 复制并填写环境变量
cp .env.example .env

# 4. 初始化数据库
npm run db:migrate

# 5. 启动开发服务器（含热重载）
npm run dev
# 服务监听在 http://localhost:8080

# 6. 客户端切换到本地测试模式：
#    Neko Status 客户端 → 设置 → 服务器地址 → 切换到「本地测试」→ http://localhost:8080
```

## 六、生产部署要点

- 使用 **HTTPS**（客户端默认连接 `https://`）
- 推荐将 `uploads/` 目录挂载到云存储（OSS / S3 / R2）
- 建议在 nginx/caddy 前置反向代理，设置上传大小限制（默认 10MB）
- WebSocket 需确保反向代理开启 `Upgrade` 头透传
- 参考 [WEB_NF2 部署文档](../../../WEB_NF2/服务端部署文档（务必查看阅读！！！）/) 的环境配置章节
